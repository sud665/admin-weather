# 위시켓 포트폴리오 & 지원서 자동 생성 프롬프트

## 사용법
1. 아래 [프로젝트 정보 입력] 섹션을 채운다
2. 전체 프롬프트를 Claude에 붙여넣는다
3. 포트폴리오 등록용 + 프로젝트 지원용 콘텐츠가 한 번에 생성된다

## [출력 형식] 아래 형식에 맞춰 작성해주세요.

### ========== Part 1: 포트폴리오 등록용 ==========

#### 1-1. 포트폴리오 제목
- 핵심을 나타내는 한 줄 제목
- 형식: "{프로젝트 핵심 키워드} - {결과물 유형} {기술적 차별점}"
- 예시: "기후변화 피해비용 시각화 플랫폼 - 531,441가지 조합 인터랙티브 대시보드 개발"

#### 1-2. 업무 범위 및 분야 선택 가이드
- 업무 범위: [입력된 내용 기반 체크 항목]
- 카테고리: [입력된 내용 기반 선택 항목]
- 포트폴리오 분야: 위시켓 추천 분야 중 가장 적합한 것 2~3개 추천
- 관련 기술: 쉼표로 구분된 기술 목록

#### 1-3. 프로젝트 배경 (위시켓 형식)
```

- (3가지 bullet)

## [작성 규칙]

1. **구체성**: 추상적 표현 대신 수치, 기술명, 기능명을 사용
   - X: "효율적인 시스템 구축"
   - O: "CSV 업로드/검증/미리보기 기능으로 비개발자도 데이터 관리 가능"

2. **간결성**: 위시켓 글자 제한 준수 (성과 설명 120자, 지원내용 2000자)

3. **연결성**: 포트폴리오의 경험이 지원 프로젝트에 왜 도움이 되는지 명확히 연결

4. **신뢰감**:
   - 진행 제안에 구체적 일정 포함
   - 주 단위 소통 방식 명시
   - 기술적 의사결정 근거 포함

5. **차별화**:
   - 단순 기능 나열이 아닌 "왜 이렇게 구현했는지" 포함
   - 문제 해결 관점으로 서술
   - 성과에 가능하면 수치 포함

6. **금지 사항**:
   - 이메일, 전화번호, 회사명 등 직거래 유도 정보 포함 금지
   - 과장된 표현 지양 ("최고의", "혁신적인" 등)
   - 검증 불가능한 수치 사용 금지
```

## 빠른 사용 예시

프로젝트 정보만 간단히 넣어도 됩니다:

```
위 프롬프트를 사용해서 아래 프로젝트의 위시켓 포트폴리오와 지원서를 작성해줘.

- 프로젝트명: 사내 문서 검색 시스템
- 한줄 설명: AI 기반 사내 문서 통합 검색 및 질의응답 시스템
- 업무 범위: 개발
- 카테고리: 웹
- 참여 기간: 2025.10 ~ 2026.01
- 내 역할: 풀스택 개발
- 문제: 사내 문서가 분산되어 검색이 어려움
- 주요 기능: RAG 기반 문서 검색, 자연어 질의응답, 문서 자동 분류, 관리자 대시보드
- 기술 스택: Next.js, TypeScript, Python, FastAPI, PostgreSQL, pgvector, OpenAI API, Tailwind CSS
- 성과: 문서 검색 시간 평균 70% 단축, 월 500건 이상 질의 처리
```
